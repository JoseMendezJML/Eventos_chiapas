# Guía de colaboración

## Flujo de trabajo

1. Actualizar la rama main.
2. Crear una rama nueva.
3. Realizar cambios en Visual Studio Code.
4. Guardar los archivos modificados.
5. Hacer commits descriptivos.
6. Subir la rama al repositorio remoto.
7. Crear Pull Request.
8. Solicitar revisión de un compañero.
9. Corregir observaciones si existen.
10. Hacer merge cuando sea aprobado.

---

## Reglas para nombrar ramas

Las ramas deben tener nombres claros y relacionados con la actividad que se realizará.

### Ejemplos:

- rama-header
- rama-eventos
- rama-contacto
- rama-estilos
- rama-script
- rama-conflicto-a
- rama-conflicto-b

---

## Reglas para escribir commits

Los mensajes de commit deben ser descriptivos y explicar claramente el cambio realizado.

### Ejemplos correctos:

- Agrega estructura del encabezado principal
- Crea formulario de contacto
- Mejora diseño de tarjetas
- Agrega validación del formulario
- Corrige enlaces de navegación

### Ejemplos incorrectos:

- cambios
- update
- prueba
- final
- cosas

---

## Reglas para Pull Requests

Cada Pull Request debe incluir:

- Título claro.
- Descripción de los cambios realizados.
- Revisión de otro compañero.
- Comentario de aprobación.
- Merge después de la aprobación.

---

## Revisión de código

El compañero revisor debe verificar:

- Que el código funcione correctamente.
- Que no existan errores visuales.
- Que los enlaces funcionen.
- Que los archivos estén correctamente organizados.
- Que el commit tenga sentido.
- Que los cambios correspondan a la rama trabajada.
